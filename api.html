Also note that on Windows, the -d argument in curl might need to be quoted in double quotes instead of single quotes. Alternatively, the /ocpu/test page can be used to explore the API.

The OpenCPU root path

The root of the OpenCPU API is dynamic. It usually defaults to /ocpu/, however system administrators can change this. Clients should make the OpenCPU server address and root path configurable. In the examples below we assume the default /ocpu/.

HTTP Methods

OpenCPU currently only uses HTTP methods GET and POST. GET is used to read a resource, and POST is to execute/call/run the resource. POST is only valid on scripts, reproducible documents and R functions.

Method Target Action Arguments Example
GET object read object control output format GET /ocpu/library/datasets/R/mtcars/json
POST object call function function arguments POST /ocpu/library/stats/R/rnorm
GET file read file - GET /ocpu/library/MASS/NEWS
GET /ocpu/library/MASS/scripts/
POST file run script control interpreter POST /ocpu/library/MASS/scripts/ch01.R
POST /ocpu/library/knitr/examples/knitr-minimal.Rmd
{% highlight bash %} #curl uses http get method by default curl https://public.opencpu.org/ocpu/library/datasets/R/mtcars/json curl https://public.opencpu.org/ocpu/library/MASS/NEWS curl https://public.opencpu.org/ocpu/library/MASS/scripts/ #curl uses http post method for -X POST or -d 'arg=postdata' curl https://public.opencpu.org/ocpu/library/MASS/scripts/ch01.R -X POST curl https://public.opencpu.org/ocpu/library/stats/R/rnorm -d 'n=10&mean=5' {% endhighlight %}

HTTP Status Codes

These are common statuscodes returned by OpenCPU that the client should be able to interpret

HTTP Code When Returns
200 OK On successful GET request Resource content
201 Created On successful POST request Output location
302 Found Redirect Redirect Location
400 Bad Request R raised an error. Error message in text/plain
502 Bad Gateway Nginx (opencpu-cache) can't connect to OpenCPU server. (admin needs to look in error logs)
503 Bad Request Serious problem with the server (admin needs to look in error logs)

The API Libraries

The following libraries hold packages and sessions that are available on the server

Path What
/ocpu/library/:pkgname/ Interfaces to R packages installed in one of the global libraries on the server.
/ocpu/user/:username/library/:pkgname/ Interfaces to R packages installed in the home library of :username.
/ocpu/tmp/:sessionid/ Interfaces temporary sessions, usually the result of function/script executions.
{% highlight bash %} #read packages curl https://public.opencpu.org/ocpu/library/MASS curl https://public.opencpu.org/ocpu/user/jeroen/somepackage #read session curl https://public.opencpu.org/ocpu/tmp/x2c5ab8d4 {% endhighlight %}

In addition, a server might have support for the following remote API's enabled:

Path What
/ocpu/cran/:pkgname/ Interfaces to the R package :pkgname that is current on CRAN.
/ocpu/bioc/:pkgname/ Interfaces to the R package :pkgname that is current on BioConductor.
/ocpu/github/:gituser/:pkgname/ Interfaces to R package :pkgname in the master branch of the identically named repository from github user :gituser.
/ocpu/gist/:gituser/:path Static files and directories located in the gist repository from github user gituser. This API does not support packages, just files (scripts, html, etc)
{% highlight bash %} #read packages curl https://public.opencpu.org/ocpu/cran/plyr/ curl https://public.opencpu.org/ocpu/github/hadley/ curl https://public.opencpu.org/ocpu/github/hadley/plyr/ #read files curl https://public.opencpu.org/ocpu/gist/hadley/ {% endhighlight %}

The R package API

Any of the libraries above that supports the /:package/ API can call the following endpoints:

Path What
../:pkgname/ Show information about this package.
../:pkgname/R/ List R objects exported by this package. See R object API.
../:pkgname/man/ List manuals (help pages) available in this package.
../:pkgname/man/:topic/:manformat Get help page about topic in output format manformat. Manual format must be one of text, html, pdf
../:pkgname/html Simulates the R-base html help pages (for backward compatibility).
../:pkgname/:path For anything else, interfaces to the file at :path in the package installation directory.
{% highlight bash %} #read package resources curl https://public.opencpu.org/ocpu/library/MASS/ curl https://public.opencpu.org/ocpu/library/MASS/R/ curl https://public.opencpu.org/ocpu/library/MASS/R/rlm/print curl https://public.opencpu.org/ocpu/library/MASS/man/ curl https://public.opencpu.org/ocpu/library/MASS/man/rlm/text curl https://public.opencpu.org/ocpu/library/MASS/man/rlm/html curl https://public.opencpu.org/ocpu/library/MASS/man/rlm/pdf #read files included with this package curl https://public.opencpu.org/ocpu/library/MASS/scripts/ curl https://public.opencpu.org/ocpu/library/MASS/scripts/ch01.R curl https://public.opencpu.org/ocpu/library/MASS/NEWS #call a function from rlm manual curl https://public.opencpu.org/ocpu/library/MASS/R/rlm -d 'formula=stack.loss ~ .&data=stackloss&psi=psi.bisquare' #run R script curl https://public.opencpu.org/ocpu/library/MASS/scripts/ch01.R -X POST #read output (replace with returned sessionid) curl https://public.opencpu.org/ocpu/tmp/x0648ec6b/R/.val/print {% endhighlight %}

The R object API

The /R API is used to read R objects, or call R functions.

Path What
../R/ List R objects in this package or session.
../R/:object Read R object in default format. If object is a function, it can be called using HTTP POST.
../R/:object/:format Retrieve an R object in a particular output format (see section on output formats).

The R session API

Sessions are currently only used to store the output of a function/script call.

Path What
/ocpu/tmp/:sessionid/ List available output for this session.
/ocpu/tmp/:sessionid/R Interfaces to the R API for R objects stored in this session.
/ocpu/tmp/:sessionid/graphics/ List graphics (plots) stored in this session.
/ocpu/tmp/:sessionid/graphics/:n/:format Retrieve plot number :n in output format :format. Format is usually one of png, pdf or svg. The :n is an integer or "last".
/ocpu/tmp/:sessionid/source Reads the input source code for this session.
/ocpu/tmp/:sessionid/stdout Shows text printed to STDOUT in this session.
/ocpu/tmp/:sessionid/console Shows the console input/output for this session (combines source and stdout)
/ocpu/tmp/:sessionid/zip Download the entire session as a zip archive.
/ocpu/tmp/:sessionid/tar Download the entire session as a gzipped tarball.
/ocpu/tmp/:sessionid/files/:path Interfaces to the file API in the working dir of the session.
{% highlight bash %} #A POST will create a temporary session #Use the returned sessionid for the subsequent calls below curl https://public.opencpu.org/ocpu/library/MASS/scripts/ch01.R -X POST #Look at console input/output curl https://public.opencpu.org/ocpu/tmp/x05b85461/console/text #We read session R objects curl https://public.opencpu.org/ocpu/tmp/x05b85461/ curl https://public.opencpu.org/ocpu/tmp/x05b85461/R/ curl https://public.opencpu.org/ocpu/tmp/x05b85461/R/dd/csv curl https://public.opencpu.org/ocpu/tmp/x05b85461/R/t.stat/print #Or even call a function curl https://public.opencpu.org/ocpu/tmp/x05b85461/R/t.stat -d 'x=[1,0,0,1,1,1,0,1,1,0]' #Download file from the working dir curl https://public.opencpu.org/ocpu/tmp/x05b85461/files/ch01.pdf #Check sessionInfo curl https://public.opencpu.org/ocpu/tmp/x05b85461/info/print {% endhighlight %}

Argument formats for R function calls (HTTP POST only)

When calling a function, we need to pass arguments. OpenCPU accepts the following types of arguments

Type What Examples
Primitives Numbers, strings, booleans 3.1415, false, "mytitle"
JSON object An array or object {"foo" : [1,2,3], "bar" : {"name" : "jerry"}}
R code plain R code paste("useR", "2013"), mtcars
File upload File argument will be temporary file path. (multipart only)
Chained Object Session ID from previous function call. x2ac58d69

When performing a fucntion call, the POST data (request body) can be formated using either multipart/form-data, application/x-www-form-urlencoded or application/json. Not every content-type supports any argument format:

Content-type Primitives JSON object R code File upload Chained object
application/x-www-form-urlencoded
multipart/form-data
application/json
{% highlight bash %} #call some functions curl https://public.opencpu.org/ocpu/library/stats/R/rnorm -d 'n=10&mean=5' curl https://public.opencpu.org/ocpu/library/graphics/R/hist -d 'x=[2,3,2,3,4,3,3]&breaks=10' curl https://public.opencpu.org/ocpu/library/graphics/R/plot -d 'x=cars&main="test"' curl https://public.opencpu.org/ocpu/library/base/R/identity -d 'x=coef(lm(speed~dist, data=cars))' {% endhighlight %} {% highlight bash %} #upload local file mydata.csv curl https://public.opencpu.org/ocpu/library/utils/R/read.csv -F "file=@mydata.csv" #replace session id with returned one above curl https://public.opencpu.org/ocpu/tmp/x067b4172/R/.val/print curl https://public.opencpu.org/ocpu/library/base/R/summary -d 'object=x067b4172' {% endhighlight %} {% highlight bash %} #post arguments in json curl http://your.server.com/ocpu/library/stats/R/rnorm \ -H "Content-Type: application/json" -d '{"n":10, "mean": 10, "sd":10}' {% endhighlight %}

JSON I/O RPC (a.k.a. Data Processing Units)

For the common special case where a client is only interested in the output data from a function call in JSON format, the HTTP POST request url can be post-fixed with /json. In this case, a successfull call will return status 200 (instead of 201), and the response body directly contains the returned object in JSON; no need for an additional GET request.

{% highlight bash %} curl http://your.server.com/ocpu/library/stats/R/rnorm/json -d n=2 [ -1.2804, -0.75013 ] {% endhighlight %}

We can combine this with the application/json request content type for full JSON RPC on R functions

{% highlight bash %} curl http://your.server.com/ocpu/library/stats/R/rnorm/json \ -H "Content-Type: application/json" -d '{"n":3, "mean": 10, "sd":10}' [ 4.9829, 6.3104, 11.411 ] {% endhighlight %}

The above request invokes the following R function call:

{% highlight r %} library(RJSONIO) args <- fromJSON('{"n":3, "mean": 10, "sd":10}', simplifyWithNames = FALSE) do.call(stats::rnorm, args) {% endhighlight r%}

Which in this case is equivalent to:

{% highlight r %} rnorm(n=3, mean=10, sd=10) {% endhighlight %}

Running scripts and reproducible documents

We can run a script by performing HTTP POST on a file. The script is interpreted according to its (case insensitive) file extension. Any HTTP POST arguments are passed on to the interpreting function. The following types are supported:

File extension Type Interpreter Arguments
file.r R script evaluate::evaluate -
file.tex latex tools::texi2pdf -
file.rnw knitr/sweave knitr::knit + tools::texi2pdf -
file.md markdown knitr::pandoc format (see ?pandoc)
file.rmd knitr/markdown knitr::knit + knitr::pandoc format (see ?pandoc)
file.brew brew brew::brew output (see ?brew)
{% highlight bash %} #run scripts which exist in packages curl https://public.opencpu.org/ocpu/library/MASS/scripts/ch01.R -X POST curl https://public.opencpu.org/ocpu/library/brew/featurefull.brew -X POST curl https://public.opencpu.org/ocpu/library/brew/brew-test-2.brew -d 'output=output.html' curl https://public.opencpu.org/ocpu/library/knitr/examples/knitr-minimal.Rmd -X POST curl https://public.opencpu.org/ocpu/library/knitr/examples/knitr-minimal.Rmd -d 'format=docx' curl https://public.opencpu.org/ocpu/library/knitr/examples/knitr-minimal.Rmd -d 'format=html' curl https://public.opencpu.org/ocpu/library/knitr/examples/knitr-minimal.Rnw #run scripts from gist curl https://public.opencpu.org/ocpu/gist/hadley/6195660/r-is-like.md -X POST curl https://public.opencpu.org/ocpu/gist/hadley/5721744/html.r -X POST {% endhighlight %}

OpenCPU Apps

OpenCPU apps are static web pages (html, css, js) which are included in an R package. They interface the R functions in this package through the OpenCPU API's. By convention, these apps are placed in the /inst/www/ directory of the R source package.

See for more information the apps page.
