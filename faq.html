What is OpenCPU?
OpenCPU is a software framework for embedded statistical computation and reproducible research. The server exposes a web API interfacing R, Latex and Pandoc. This API is used for example to integrate statistical functionality into systems, share and execute scripts or reports on centralized servers, and build R based "apps". The OpenCPU server can run either as a single-user server inside the interactive R session (for local use and development), or as a cloud server that builds on Linux and rApache.
What are OpenCPU Apps?
An OpenCPU app is an R package which includes some web page(s) that call the R functions in the package using the OpenCPU API. This makes a convenient way to develop, package and ship standalone R web applications. See the server manual for more information.
Does the OpenCPU cloud server support { Windows | OSX | RHEL | etc } ?
You are asking the wrong question. The OpenCPU cloud server is designed to run on a remote server in the cloud or virtual machine. Once installed, you will be interacting with OpenCPU mostly through the API. It not recommended to host other software on the same server as OpenCPU, as R can be greedy with resources, and it is also unwise from a security point of view.
What about security?
Security is enforced using AppArmor. AppArmor is a security module for the linux kernel and enforces mandatory access control policies on a by-process basis. These policies can easily be customized by modifying the AppArmor profiles that ship with the OpenCPU cloud server. See the server manual for more information.
Can I use OpenCPU with RStudio?
Yes! Both the OpenCPU single-user server and OpenCPU cloud server have been tested to work with RStudio 0.98 and above. See also the appendix in the server manual on this topic.
Can OpenCPU be used to build proprietary applications?
Absolutely. OpenCPU is released under the Apache2 License.
