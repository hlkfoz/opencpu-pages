About the Library

The opencpu.js JavaScript client library builds on jQuery and provides Javascript interfaces to call R from within a web page via the OpenCPU API. The library works on all modern browsers, and is a solid foundation for building R web applications.

OpenCPU Apps

An OpenCPU app is an R packages which includes web page(s) that calls the R functions in the package over the OpenCPU API. This makes a convenient way to develop, package and ship portable, standalone R web applications.

A repository of public OpenCPU apps is available at http://github.com/opencpu. These apps can be installed on any opencpu server:

{% highlight r %} #install apps 'stocks', 'markdownapp' and 'nabel' library(devtools) install_github(c("stocks", "markdownapp", "nabel"), username="opencpu") {% endhighlight %}

By convention, the app is placed in the /inst/www directory in the R package, so the app is available at /ocpu/library/app/www/ on your server:

  • https://public.opencpu.org/ocpu/library/stocks/www
  • https://public.opencpu.org/ocpu/library/markdownapp/www
  • https://public.opencpu.org/ocpu/library/nabel/www

Ajax and jQuery

The latest version of opencpu.js is available from github: https://github.com/jeroenooms/opencpu.js.

The library depends on jQuery. This means jQuery must be included in your web page before opencpu.js. Your application code must be inluced after opencpu.js.

{% highlight html %} {% endhighlight %}

It is recommended to ship a copy of the opencpu.js library with your application or website (as opposed to hotlinking it from some public location). This because the JavaScript library is in active development (0.x version) and the latest version might (radically) change at any time. Shipping a version of opencpu.js with your app prevents it from breaking with upstream changes in the library. Also it is practical both for development and deployment if your app can work offline.

Cross Domain / CORS

The recommended way to ship OpenCPU apps is by including the web pages with the R package. This results in a standalone application, that is easy to ship and deploy, guarantees that the version of front-end and R code are in sync, and automatically resolves R dependency packages when installed. However it is also possible to use the opencpu.js library from an external site that is itself not hosted on OpenCPU.
